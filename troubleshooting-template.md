# 깃허브에 올리는 법 (코딩 몰라도 되는 버전)

이 폴더를 깃허브에 올리는 가장 쉬운 방법입니다. git 명령어 없이 웹 브라우저만 사용합니다.

## 1. 레포지토리 만들기 (5분)

1. https://github.com 가입 → 로그인
2. 우측 상단 `+` → **New repository**
3. Repository name: `sse-engineering-notes`
4. **Public** 선택 (공개가 핵심!)
5. "Add a README file" 체크는 **하지 않음** (우리가 만든 걸 올릴 거라서)
6. **Create repository** 클릭

## 2. 파일 올리기 (5분)

1. 생성된 레포 화면에서 **uploading an existing file** 링크 클릭
2. 이 폴더(`sse-engineering-notes`) 안의 파일과 폴더를 전부 드래그&드롭
3. 하단 Commit message에 `첫 업로드` 입력 → **Commit changes**

끝. 이제 `https://github.com/본인아이디/sse-engineering-notes` 가
당신의 공개 폴더입니다. README.md가 자동으로 대문에 표시됩니다.

## 3. 다음 글 쓰는 법

- 레포 화면에서 폴더 진입 → **Add file → Create new file**
- 파일명 입력 (예: `prisma-access/2026-07-tunnel-flap.md`)
- `templates/troubleshooting-template.md` 내용 복사해서 채우기
- Commit changes

## 4. 나중에 (글 10개 쌓이면)

MkDocs Material을 붙여서 `본인아이디.github.io` 웹사이트로 전환.
그때 다시 물어보세요 — 단계별로 안내해 드립니다.

## 주의사항

- 올리기 전 항상 확인: 고객명, 실제 IP, 호스트명, 내부 문서 캡처 없음?
- 회사 NDA/보안정책 범위 확인 (일반화된 기술 지식 수준만)
- 이 파일(HOW-TO-UPLOAD.md)은 본인용이니 올리지 않아도 됩니다
